# Cursor 向け説明書 — このフォルダの使い方

このフォルダは **YouTube Shorts 多アカウントアップローダー** のプロジェクトです。  
**Cursor でこのフォルダを開くだけで**、AI がプロジェクトの内容を理解し、設定や起動方法を案内できます。

---

## このフォルダを Cursor で開くと

- **`.cursor/rules/`** にプロジェクト用のルールが入っています。
- Cursor の AI は「このプロジェクトは何か」「どう起動するか」「どこに何があるか」を自動で参照します。
- **何か聞くだけで**、起動方法・設定・エラー対処などを案内してもらえます。

---

## 受け取った人がやること（初回だけ）

1. **このフォルダを Cursor で開く**  
   Cursor で「ファイル → フォルダを開く」→ このフォルダを選択。

2. **AI に聞く（例）**  
   - 「このプロジェクトの起動方法を教えて」
   - 「初回セットアップの手順を教えて」
   - 「config と client_secrets の用意の仕方を教えて」

3. **実行する場所**  
   - フォルダのパスに **日本語（マイドライブなど）が含まれる場合は、このフォルダを C:\yt-shorts-app など **英数字だけのパス** にコピーしてから実行する。
   - ターミナルで `cd （フォルダのパス）` → `python run_app.py`。ブラウザで http://localhost:8501 を開く。

4. **アプリ内で完結すること**  
   - config の自動作成、client_secrets のアップロード、アカウント追加、キュー追加、予約投稿、行削除などは **すべて Web UI から** 行えます。必要に応じて AI に「キューに動画を追加する方法は？」などと聞けばよい。

---

## 配布する人へ（このフォルダを渡す前に）

- この **配布用** フォルダには、**アプリの全ファイル** と Cursor 用ルール（`.cursor/rules/`）・この説明書がすでに入っています。
- **配布するとき** は、**accounts フォルダ内の token** と **secrets/client_secrets.json** を除いた状態で、**フォルダ一式を ZIP で渡す** とよいです。
- 受け取った人は「ZIP を解凍 → そのフォルダを Cursor で開く」だけで、すぐ使える状態になります。

---

## まとめ

| やりたいこと | どうするか |
|--------------|------------|
| プロジェクトのことを知りたい | このフォルダを Cursor で開き、AI に「起動方法は？」「セットアップ手順は？」と聞く |
| 実際に動かしたい | フォルダを英数字のパスに置き、`python run_app.py` で起動 → ブラウザで http://localhost:8501 |
| 設定・アカウント・キュー | アプリの Web UI（はじめての方・アカウント・キュー・設定）で操作。不明点は AI に聞く |

このフォルダを Cursor に「投げる」だけで、AI がこのプロジェクトを理解して案内できる状態になっています。
